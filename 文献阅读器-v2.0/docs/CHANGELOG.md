## v2.0 — 批判性精读版 (2026-05-07)

### 🎯 版本定位升级
从27章分散式checklist重构为**17章清晰结构**，批判性分析维度合并为「综合批判性讨论」一个完整章节，新增综述Agent写作参考模块，去掉图片嵌入。

### ✨ 结构变化

**27章 → 17章：**
- 分散的8个批判性维度（定量基准对比、因果链验证矩阵、统计严谨性评估、效应量解读、证据强度评估、数据交叉验证、替代解释、领域定位、可复现性检查、未解答开放问题、外推风险评估、领域路线图影响评估）合并为2个章节：
  - **深度分析**（第9章）— 保留定量基准对比、机制解析、统计严谨性
  - **综合批判性讨论**（第10章★新增）— 合并证据强度、数据交叉验证、替代解释、领域定位、可复现性、未解答开放问题，并新增「创新性·重要性·可行性深度思考」

### ★ 新增内容
1. **综述Agent写作参考（第12章）**
   - 关键论证引用：1-3段改写式论点，附带效应量和关键数据，非原文摘抄
   - 文献贡献定位：适合/不适合的综述场景，避免误引用
2. **创新性·重要性·可行性深度思考**
   - 纳入综合批判性讨论末尾，从三个维度综合评判论文价值

### 🔧 调整
- 关键结果章节**去掉图片嵌入**，改为文字指明Figure对应关系
- 图片单独存放于 `figures/` 目录
- 关键词从「教授级」调整为「批判性精读」

### 📏 文件规模
报告结构从27章精简为17章，内容不减反增（新增综述参考模块）

## v1.4 (2026-05-06)

### ✨ 新增功能

#### 1. YAML frontmatter 结构化元数据（★综述Agent关键升级）
- 报告顶部必须包含机器可解析的YAML frontmatter元数据块
- 字段包括：paper_id, year, journal, research_type, field/subfield, cell_types, animal_model, key_techniques, effect_size_summary, tags（concepts/methods/findings三级标签）
- 综述Agent可直接用 `yaml` 库解析，无需从正文中提取

#### 2. 关键数据提取表（★综述Agent关键升级）
- 报告末尾（引用之前）新增标准化数据提取表
- 含指标、对照组、实验组、效应量、统计显著性、样本量六列
- 支持综述Agent做定量对比和效应量汇总
- 严格原则：只填论文明确给出的数据，不推测

#### 3. 创新层级评估
- 每条创新点必须标注层级标签：【颠覆性创新】/【重要机制发现】/【增量改进】/【方法学创新】/【应用创新】
- 综述Agent可据此在讨论中分配权重

#### 4. 报告锚点ID
- 章节使用 `{#section-id}` 锚点，便于综述Agent精确定位

### 📋 新增/修改报告章节一览
| 章节 | 类型 | 说明 |
|------|------|------|
| 结构化元数据 | ★新增 | YAML frontmatter机器可解析元数据 |
| 关键数据提取表 | ★新增 | 标准化定量数据表，支持Meta分析 |
| 创新层级评估 | ★新增 | 颠覆性/机制/增量/方法/应用五级标签 |

### 🔧 优化
- 质量要求新增4条：创新分级、YAML frontmatter、数据提取表、锚点ID
- AI工作流程新增：报告头部YAML frontmatter生成说明+示例、关键数据提取表生成说明+示例
- 报告结构表新增3行
- 版本号 v1.3 → v1.4

## v1.3 (2026-05-06)

### ✨ 新增功能

#### 1. 批判性分析体系（★核心升级）
- **证据强度评估**：新增报告章节，对论文核心结论进行强/中/弱三级证据分级，避免过度解读
- **数据交叉验证**：新增报告章节，要求检查不同指标间的定量关系是否自洽（如EV颗粒数增幅 vs 蛋白总量增幅），发现并标注数据矛盾
- **替代解释**：新增报告章节，主动探讨作者未排除的其他科学解释
- **领域定位**：新增报告章节，将研究放在领域发展脉络中评估是突破性进展还是增量改进
- **可复现性检查**：新增报告章节，检查数据/代码公开情况、关键试剂货号和实验设计完整性

#### 2. AI内部工作流程强化
- 深度阅读理解步骤增加 **★批判性审视★**：要求AI主动寻找证据链中的逻辑漏洞和未被排除的替代假说
- 新增 **数据交叉验证** 步骤：检查跨指标一致性，发现矛盾需突出标注
- 新增 **证据强度分级评估** 步骤：按强/中/弱三级评估核心结论的可信度

#### 3. 图片处理增强
- 支持**子图级命名和引用**：Figure包含多个子图（如Figure 3A-M）时可拆分到子图级别命名
- 支持**图片类型标签**：文件名后可附加类型标签，如 `Figure1_示意图.png`、`Figure2_统计图.png`、`Figure3E_电镜.png`

### 📋 新增报告章节一览

| 章节 | 类型 | 说明 |
|------|------|------|
| 证据强度评估 | ★新增 | 样本量、统计方法、效应量的系统评估 |
| 数据交叉验证 | ★新增 | 跨指标定量关系自洽性检查 |
| 领域定位 | ★新增 | 研究在领域发展中的定位评估 |
| 可复现性检查 | ★新增 | 数据/代码/试剂的信息核查 |
| 替代解释 | ★新增 | 作者未排除的其他科学解释探讨 |

### 🔧 优化
- 报告结构表更新：关键结果章节支持子图级引用
- 质量要求新增3条：批判性思维、数据一致性、子图级引用
- 输出标准新增5项：批判性数据交叉验证、证据强度分级、替代解释、领域定位、可复现性
- 图片命名规则增加子图支持和类型标签

## v1.2 (初始版本)

### ✨ 核心功能
- 文献解析（PDF/Word/TXT格式）
- 阅读辅助（高亮、笔记、术语解释）
- 文献分析（引用关系、研究趋势、方法提取）
- 文献管理（分类、检索、导出、Obsidian集成）
- 深度阅读笔记生成

### 📋 报告章节（v1.2标准）
| 章节 | 说明 |
|------|------|
| 核心信息 | 标题、作者、年份、期刊、DOI、关键词 |
| 原文摘要翻译 | 完整中文摘要翻译 |
| 创新点 | 逐条列出核心创新 |
| 一句话总结 | 概括全文核心贡献 |
| 研究问题 | 核心科学问题 + 研究动机 |
| 数据与任务定义 | 实验数据来源 + 验证任务列表 |
| 方法主线 | 分阶段方法流程 |
| 关键结果 | 数据表格 + 图片引用 |
| 深度分析 | 技术路线 + 方法对比 + 机制解析 |
| 局限性评估 | 逐条列出并客观分析 |
| 研究启发 | 对后续研究的启发 |
| 引用 | 标准引用格式 |

### 🏗 技术架构
- Python脚本体系：extract_pdf_assets.py, analyze_and_report.py, write_obsidian_note.py 等
- 依赖：PyMuPDF、Pillow、deep_translator
- 与Obsidian集成能力
- Windows编码兼容性处理
